TransitOps
Software Requirements Specification (Hackathon
Edition)
1. Introduction
TransitOps is a centralized transport operations platform to manage vehicles, drivers, dispatch,
maintenance, fuel, expenses, and analytics.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
2. Purpose
Replace spreadsheets with a secure web application that enforces business rules and provides
operational visibility.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
3. Scope
Authentication, RBAC, Dashboard, Vehicle Registry, Driver Management, Trip Management,
Maintenance, Fuel & Expense Management, Reports.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
4. Stakeholders
Fleet Manager, Dispatcher, Safety Officer, Financial Analyst, System Administrator.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
5. User Roles
Fleet Manager manages vehicles and maintenance. Dispatcher creates and dispatches trips.
Safety Officer monitors licenses and compliance. Financial Analyst reviews costs and ROI.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
6. Functional Requirements
Authentication, CRUD operations, trip lifecycle, automatic status updates, maintenance workflow,
fuel logs, expense tracking, dashboards, CSV export.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
7. Non-Functional Requirements
Responsive UI, role-based security, auditability, scalability, maintainability, availability,
performance.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
8. Business Rules
Unique registration numbers; no dispatch of retired/in-shop vehicles; expired/suspended drivers
cannot be dispatched; no double assignment; cargo must not exceed capacity;
dispatch/completion/cancellation automatically update statuses.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
9. Database Design
Entities: Users, Roles, Vehicles, Drivers, Trips, MaintenanceLogs, FuelLogs, Expenses.
Relationships: Trip references Vehicle and Driver; Maintenance/Fuel/Expense reference Vehicle.
Use Cases
Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
10. API Specification
POST /login, GET/POST/PUT/DELETE /vehicles, /drivers, /trips, /maintenance, /fuel, /expenses,
GET /dashboard.

Typical flow: Login → Manage master data → Create Trip → Dispatch → Complete → Maintenance
→ Reports.
